Target Users

Major global retailers

Technologies / Skills

Children, Inc. Service Learning

Client

Children Inc, Service Learning

Title

Web Developer/Designer

Project

Design and add functionality by adding a large portal to a pre-existing website design with a pre-selected theme.

The purpose of the portal to the website is for viewing and downloading PDF lesson plans.

NOTE: I designed and added the portal and not the pre-existing website. I did make design changes as noted in this summary.


Solution

Develop a lesson plan portal and CMS content update method for the pre-exiting website. The portal will allow teachers and students to easily find and download PDF lesson plans across four subject areas and all grade levels.

Target User

Teachers and Students in the Northern Kentucky and Cincinnati Area.

Technologies

  • WordPress/Plugins
  • UX Design
  • HTML
  • CSS
  • PhotoShop

My Process / What I Did

  • Define purpose/goals of project with client (program director).
  • Determine all technologies needed to complete project.
  • Research methods, coding, and plugins to meet project goals.
  • Use UX best practices / user scenarios to decide best user-friendly navigation.
  • Sketch potential solutions.
  • Develop and test sample portal section for usability and complete portal structure and plugins needed.
  • Build full menu and add page/plugin integration with simple CMS content update method.
  • Add lesson plans to populate all grade levels for four subject areas.
  • Add and setup Gravity forms
  • Performed image editing/cropping/enhancing.
  • Add styling to portal and forms with custom CSS.
  • Test and proof new portal and go live.
  • Create client tutorials for CMS content update methods via documentation and video screen capture examples.

Example links shown can display lesson plan PDF's.

The Song Wars

Client

TheSongWars.com (Lawrence Jones - Site Owner)

Title

Web Developer and Designer

Project

Create full life-cycle product with WordPress CMS for entertainment website that allows users to listen and vote on their favorite songs.

Solution

Create a CMS WordPress website that allows users to easily login with their social media accounts, listen to song battles, and vote on their favorite song. Also provide a method for front end users to submit songs and for the site owner to add and control song battles.

Target User

Majority of users/audience is between the ages of 19 and 34.

Technologies / Skills

  • WordPress/Plugins
  • HTML
  • CSS
  • JavaScript
  • PhotoShop
  • Social Login Integration API's
  • Graphic Design
  • User Scenarios / Task Flows
  • Web Hosting Server Setup

My Process / What I Did

  • Client consultation / define purpose and goals of website.
  • Wrote contract with legal protection for both parties with itemized work to be performed and price quote to send with billing invoice.
  • Determine site structure and technologies needed.
  • Research entertainment websites and decide on theme for website.
  • Collected Assets from Client
  • Setup self-hosted WordPress website
  • Determed best navigation and dynamic content structure
  • Researched, select, and setup multiple advanced plugins for dynamic content and functionality.
  • Setup plugin that allows custom JavaScript and CSS to assigned pages.
  • Customized basic audio player and voting plugins using JavaScript for controlled display sequence.
  • Integrated plugins with pages for best CMS content update methods.
  • Added song up-loader capability.
  • Added and setup third party service with Social Login API's
  • Setup Google Analytics
  • Tested and Proofed website
  • Went live with website.
  • Created CMS content update documentation and provided client training.

Social Logins with API Services

Setup API services with all major social media sites to allow easy user logins.

Dynamically Displayed Song Battle Links

Used page / plugin integration that allows easy CMS content updates/additions. New song battles added in CMS can be assigned to a category that will automatically display song battle links.

Song Battle User Interface

Customized audio player and voting plugin to display in sequence for best usability. Listen and vote sessions are timed to display 30 second samples and progress from one song to another. CSS was used to style UI and remove unneccessary elements making it easier to guide the user through the process.

Song Battle Voting UI

After songs play, the audio panel changes into a voting panel via JavaScript.
